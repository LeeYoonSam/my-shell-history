## [6. 파일 복사하고 용량을 초과하면 오래된 폴더부터 삭제하는 스크립트](file-copy-delete-limit-exceeded.sh)

### 기록
- `cp`: 파일 복사 
- `du`: 디스크 사용량 통계, `-sh` 옵션으로 지정된 각 파일에 대한 항목 표시(-s), 사람이 읽을수 있도록 출력(-h)
- `cut`: 자르고 첫번째 필드(-f1), 구분자 기준으로 자르기(-d)
- `find`: 찾기
    - `-type d`: 지정된 타입인지 검사 b, c, d(irectory), f, l, p, s
    - `-maxdepth n`
        - 명령줄 인수 아래 최대 n개의 디렉토리 수준으로 내려갑니다.  최대 깊이를 기본으로 지정하면 일반적으로 평가되지 않는 표현식이라도 전체 표현식에 적용됩니다.  "-maxdepth 0"은 전체 검색을 명령줄 인수로 제한합니다.
     - `-mindepth n`
        - n보다 작은 수준에서는 테스트나 작업을 적용하지 않습니다. -mindepth 기본값을 지정하면 일반적으로 평가되지 않더라도 전체 식에 적용됩니다.  "-mindepth 1"은 명령줄 인수를 제외한 모든 인수를 처리합니다.
- `exec`: 
    - 유틸리티라는 이름의 프로그램이 종료 상태로 0 값을 반환하면 참입니다.  
    - 선택적 인수를 유틸리티에 전달할 수 있습니다.  표현식은 세미콜론(";")으로 종료해야 합니다.  
    - 셸에서 find를 호출하는 경우 셸에서 제어 연산자로 취급하는 경우 세미콜론으로 따옴표를 제어 연산자로 취급하는 경우 세미콜론을 인용해야 합니다.  
    - 유틸리티 이름이나 인수에 "{}" 문자열이 나타나면 현재 파일의 경로명으로 대체됩니다.  
    - 유틸리티는 찾기가 실행된 디렉토리에서 실행됩니다.  
    - 유틸리티와 인수는 셸 패턴 및 구조체의 추가 확장에 확장되지 않습니다.
- `rm`: 디렉토리 삭제

## [11. 디렉토리 파일 정리](organizing-files.sh)

디렉토리에서 파일을 정리하는 프로세스를 자동화하는 데 도움이 되는 셸 스크립트를 작성하세요. 

이 스크립트는 사용자에게 디렉토리 경로를 입력하라는 메시지를 표시한 다음 유형별(예: 한 폴더의 모든 이미지 파일, 다른 폴더의 모든 문서 파일), 크기별(예: 한 폴더의 특정 크기 이상의 모든 파일) 또는 수정 날짜별(예: 한 폴더의 지난 주에 수정된 모든 파일)로 파일을 정리할 수 있는 옵션을 제공해야 합니다.

### 기록
- `read` 명령을 사용하여 사용자에게 디렉터리 경로를 묻는 메시지를 표시한 다음 대/소문자 문을 사용하여 유형, 크기 또는 수정한 날짜별로 파일을 정리할 수 있는 옵션을 제공합니다. 
- 정리 프로세스가 완료되면 사용자에게 원래 디렉터리에 남아 있는 파일을 삭제할 수 있는 옵션도 제공합니다.
- `stat`: 파일 상태 표시
    - `$(stat -f %z "$path")` : $path 에 해당하는 파일 크기(bytes) 정보

### Conventions
- `cd ... || exit` 로 cd 실패시에 대한 안정성을 확보
    - [SC2164 – ShellCheck Wiki](https://www.shellcheck.net/wiki/SC2164)
- `read` 사용시에 `read -r` 을 기본적으로 사용
    - `-r`: 이 옵션을 지정하면 백슬래시가 이스케이프 문자로 작동하지 않습니다.
    - [SC2162 – ShellCheck Wiki](https://www.shellcheck.net/wiki/SC2162)

### 사용방법
```sh
./organizing-files.sh
Enter directory path: /Users/ys/Downloads/organize-test2
Select organization option:
1. Organize by file type
2. Organize by file size
3. Organize by date modified
Enter option number: 2
Enter file size (in bytes): 200000
Delete remaining files in original directory? (y/n)n
Organization complete!
```
- `directory path` 를 절대 경로로 입력해야함

## [12. 디렉토리의 파일 압축(format 선택)](file-compressing-with-format.sh)

### 기록
- `tar`:
    - tar는 여러 개의 파일이나 디렉토리를 하나로 묶어 압축하는 명령어이며, `tape archive`의 약자입니다. 주로 리눅스나 유닉스 시스템에서 사용되며, 여러 개의 파일을 하나로 묶어서 전송하거나 백업할 때 자주 사용됩니다. 
    - tar 명령어에는 압축할 파일이나 디렉토리를 지정하고, 옵션을 추가하여 압축 방식이나 압축된 파일 이름 등을 지정할 수 있습니다. 
    - 일반적으로 `gzip`이나 `bzip2`와 같은 압축 프로그램과 함께 사용하여, `.tar` `.gz` `.tar` `.bz2` 와 같은 형식으로 파일을 압축합니다. tar의 주요 옵션에는 `-c, -x, -f, -v, -z, -j` 등이 있으며, 이 옵션들을 조합하여 다양한 기능을 수행할 수 있습니다.
- `select`: 
    - select는 셸 스크립트에서 옵션을 선택하는 메뉴를 만들 수 있는 셸 내장 명령어입니다. 
    - 이 명령은 사용자에게 선택 목록에서 옵션을 선택하라는 메시지를 표시하고 입력을 기다립니다. 
    - 사용자의 선택은 변수에 저장되어 스크립트에서 추가 작업을 수행하는 데 사용할 수 있습니다. 
    - select는 대화형 셸 스크립트를 만들고 사용자 입력을 자동화하는 데 유용합니다.

## [18. HTTP 상태 확인](website_health_checker.sh)
이 프로젝트의 목표는 몇 가지 일반적인 테스트를 수행하여 웹사이트의 상태를 검사하는 셸 스크립트를 만드는 것입니다. 

이 스크립트는 URL을 인수로 받아 다음과 같은 검사를 수행합니다:

1. HTTP 상태 코드 확인: 스크립트는 curl 명령을 사용하여 주어진 URL의 HTTP 상태 코드를 가져옵니다. 상태 코드가 200이면 "HTTP status: OK"가 출력됩니다. 상태 코드가 200이 아니면 "HTTP status: ERROR"를 출력합니다.

2. 끊어진 링크 확인: 스크립트는 wget 명령을 사용하여 지정된 URL의 HTML 콘텐츠를 다운로드한 다음 grep 명령을 사용하여 404 상태 코드를 반환하는 링크를 검색합니다. 끊어진 링크가 발견되면 "깨진 링크 발견: <깨진 링크 수>"를 출력합니다. 끊어진 링크가 발견되지 않으면 "깨진 링크를 찾을 수 없음"이 출력됩니다.

3. SSL 인증서 만료 확인: 스크립트는 openssl 명령을 사용하여 주어진 URL에 대한 SSL 인증서의 만료 날짜를 확인합니다. 인증서가 여전히 유효한 경우 "SSL 인증서: OK"가 출력됩니다. 인증서가 만료되었거나 30일 이내에 만료될 경우 "SSL 인증서: 곧 만료됨"이 출력됩니다. 

### 기록
- `curl`: curl은 HTTP, FTP, SMTP 등 다양한 프로토콜을 사용하여 서버와 데이터를 주고받는 데 사용되는 명령줄 도구입니다. "클라이언트 URL"의 약자로 웹 애플리케이션 및 서비스를 테스트하고 디버깅하는 데 널리 사용됩니다.
    - 일반적으로 사용되는 옵션으로는 자동 모드를 위한 -s, 리디렉션을 따르기 위한 -L, 헤더를 지정하기 위한 -H, 요청 본문에서 데이터를 보내기 위한 -d, 응답을 파일로 출력하기 위한 -o 등이 있습니다.

    - -s: 이 옵션은 "침묵"을 의미하며 curl이 진행률 표시기나 오류 메시지를 표시하지 않도록 합니다. 리디렉션을 숨기기 위해 -L 옵션과 함께 자주 사용됩니다.

    - -L: 이 옵션은 "위치"를 나타내며 curl이 HTTP 리디렉션을 따르도록 지시합니다. 웹사이트가 이동했거나 이름이 변경되었을 때 유용합니다.

    - -w: 이 옵션은 "쓰기"를 의미하며 curl의 출력 형식을 정의하는 데 사용됩니다. 기본적으로 curl은 가져온 콘텐츠를 표준 출력으로 인쇄합니다. w를 사용하면 출력 형식을 지정할 수 있습니다.

    - 예제 명령 curl -sL -w "%{http_code} %{url_effective}\\n" -o /dev/null <url>에서 -w는 출력 형식을 HTTP 응답 코드와 (다음 리디렉션 이후) 유효 URL로 구분하여 새줄 문자로 정의하는 데 사용됩니다. -o /dev/null은 출력을 삭제하는 /dev/null로 출력을 보내도록 curl에 지시하고, <url>은 가져올 URL을 지정합니다.

    - 따라서 이 명령은 진행률이나 오류 메시지를 표시하지 않고 HTTP 상태 코드와 웹 페이지의 유효 URL을 검색하여 /dev/null로 출력을 보냅니다.

- `wget` 명령은 인터넷에서 파일을 다운로드하는 데 널리 사용되는 도구입니다. 
    - r 옵션은 wget이 링크를 재귀적으로 따라가서 지정된 URL에서 모든 파일을 다운로드하도록 지정합니다. 
    - l 옵션은 최대 재귀 깊이를 지정합니다.
    - wget -r -l 0 <url> 명령에서 -l 0 옵션은 wget이 초기 URL보다 더 이상 재귀하지 않아야 함을 의미합니다. 즉, 초기 URL에 직접 연결된 파일만 다운로드됩니다.

- date: Linux/유닉스 시스템의 날짜 명령은 시스템 날짜와 시간을 표시하고 조작하는 데 사용됩니다. 현재 날짜와 시간을 인쇄하거나 특정 날짜와 시간을 설정하는 데 사용할 수 있습니다.
    - ```$(date -v -30d +%s)``` 30일전 날짜를 초 형식으로 표시
    - 포맷 지정: `date +"%Y-%m-%d %H:%M:%S"`
    - UTC format : `date -u`

### 참고
- [Date command in Bash](https://linuxhint.com/date-command-bash/)

## [21. 파일 암호화/복호화](./file_encrypt_descrypt.sh)
이 bash 스크립트를 통해 사용자는 암호 기반 대칭 암호화 알고리즘을 사용하여 파일을 암호화하고 해독할 수 있습니다. 스크립트는 `OpenSSL` 라이브러리를 활용하여 암호화 및 암호 해독 작업을 수행합니다.

스크립트에서 사용자가 선택 사항으로 `e`를 입력하면 스크립트는 무작위로 생성된 솔트와 지정된 비밀번호를 사용하여 CBC 모드에서 AES-256을 사용하여 입력 파일을 암호화합니다. 그런 다음 출력은 확장자가 `.enc`인 파일에 저장됩니다.

사용자가 선택 사항으로 `d`를 입력하면 스크립트는 동일한 암호화 알고리즘, 솔트 및 비밀번호를 사용하여 입력 파일을 복호화합니다. 그런 다음 출력은 확장자 `.dec`를 가진 파일에 저장됩니다.

### 기록
- `openssl`: OpenSSL은 다양한 암호화 작업을 수행하는 데 사용되는 명령줄 도구입니다. 여기서는 파일을 암호화하거나 해독하는 데 사용됩니다.
- `aes-256-cbc`: 사용할 암호화 알고리즘을 지정합니다. 이 경우 CBC 모드에서 AES-256입니다.
- `-a`: 이 옵션은 출력이 base64로 인코딩되도록 지정합니다.
- `-salt`: 이 옵션은 암호화 프로세스에 임의의 솔트를 추가하여 크래킹하기 어렵게 만듭니다.
- `-in`: 입력 파일 경로를 지정합니다.
- `-out`: 출력 파일 경로를 지정합니다.
- `-pass`: 파일을 암호화하거나 해독하는 데 사용되는 비밀번호를 지정합니다. 이 경우 비밀번호는 pass: 접두사와 함께 인수로 전달됩니다.

## [23. 로그 파일 분석](./file_encrypt_descrypt.sh)
요약 설명:
- 이 스크립트는 로그 파일을 분석하여 특정 이벤트 또는 패턴의 발생에 대한 인사이트를 제공합니다. 
- 로그 파일 내에서 추세, 오류 또는 이상 징후를 식별하고 추가 분석을 위해 관련 정보를 추출하는 데 도움이 됩니다.

요구 사항:
1. 스크립트는 로그 파일 경로를 명령줄 인수로 받아들여야 합니다.
2. 로그 파일 내에서 특정 이벤트 또는 패턴을 검색할 수 있는 옵션을 제공해야 합니다.
3. 스크립트는 발견된 각 이벤트 또는 패턴에 대한 총 발생 횟수를 표시해야 합니다.
4. 또한 상위 N개를 포함하여 가장 빈번하게 발생하는 이벤트 또는 패턴의 요약도 표시해야 합니다.
5. 사용자는 표시할 상위 발생 횟수를 지정할 수 있어야 합니다.
6. 스크립트는 유연성을 위해 대소문자를 구분하지 않는 검색을 지원해야 합니다.
7. 오류 처리는 유효한 로그 파일 경로를 확인하고 잠재적인 오류를 원활하게 처리하기 위해 구현되어야 합니다.

### 명령어 설명
- `grep -i -c "$pattern" "$log_file"`: 이 명령은 로그 파일에서 대소문자를 구분하지 않고 지정된 패턴을 검색하고 발생 횟수를 계산합니다.
- `grep -i -o "$search_pattern" "$log_file" | sort | uniq -c | sort -rn | head -n "$num_top_occurrences"`: 
    - 이 명령은 로그 파일에서 지정된 패턴을 검색
    - 일치하는 줄을 추출하여 정렬
    - 발생 횟수
    - 내림차순으로 정렬
    - 상위 N개의 발생 횟수를 표시
